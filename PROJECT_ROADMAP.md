## 🛠️ 开发流程规范

### 每个功能的开发流程

```
需求 → Brainstorming → Design → Plan → Implementation → Testing → Documentation → Merge
```

### 文档位置

```
docs/
├── brainstorming/    ← 需求分析、技术选型
│   └── YYYY-MM-DD-功能名.md
├── designs/          ← 架构设计、数据模型
│   └── YYYY-MM-DD-功能名.md
├── plans/            ← 实现计划、步骤拆解
│   └── YYYY-MM-DD-功能名.md
└── sql/              ← 数据库脚本
    └── YYYY-MM-DD-功能名.sql
```

### Git提交规范

```bash
# 功能提交
git commit -m "feat: 实现[功能名] - [具体描述]"

# 文档提交
git commit -m "docs: 添加[模块]文档 - [具体内容]"

# Bug修复
git commit -m "fix: 修复[问题] - [具体描述]"

# 性能优化
git commit -m "perf: 优化[模块]性能 - [具体改进]"
```
