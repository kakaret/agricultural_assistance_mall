## 数据模型

### chat_session（聊天会话）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT AUTO_INCREMENT | 主键 |
| customer_id | BIGINT NOT NULL | 买家ID，FK → user.id |
| merchant_id | BIGINT NOT NULL | 商家ID，FK → user.id |
| product_id | BIGINT NULL | 关联商品ID（从哪个商品发起） |
| status | TINYINT DEFAULT 1 | 0关闭 1开启 |
| last_message_time | TIMESTAMP NULL | 最后消息时间，用于排序 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**索引**: `idx_customer(customer_id)`, `idx_merchant(merchant_id)`, `uk_customer_merchant(customer_id, merchant_id)` UNIQUE

**设计说明**: 唯一索引保证同一买家和商家只有一个会话。再次发起时复用已有会话，更新 product_id。

### chat_message（聊天消息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT AUTO_INCREMENT | 主键 |
| session_id | BIGINT NOT NULL | 会话ID，FK → chat_session.id |
| sender_id | BIGINT NOT NULL | 发送者ID |
| sender_role | VARCHAR(20) NOT NULL | CUSTOMER / MERCHANT / SYSTEM |
| content | TEXT NOT NULL | 消息内容（文字或图片URL） |
| content_type | VARCHAR(10) DEFAULT 'TEXT' | TEXT / IMAGE |
| is_read | TINYINT DEFAULT 0 | 0未读 1已读 |
| created_at | TIMESTAMP | 创建时间 |

**索引**: `idx_session_time(session_id, created_at)`, `idx_sender(sender_id)`

### auto_reply_rule（自动回复规则）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT AUTO_INCREMENT | 主键 |
| merchant_id | BIGINT NOT NULL | 商家ID |
| keyword | VARCHAR(100) NOT NULL | 触发关键词 |
| reply_content | TEXT NOT NULL | 回复内容 |
| priority | INT DEFAULT 0 | 优先级（大的优先匹配） |
| status | TINYINT DEFAULT 1 | 0禁用 1启用 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**索引**: `idx_merchant_status(merchant_id, status)`

### ER 关系

```
user (1) ──< chat_session (N)  [customer_id]
user (1) ──< chat_session (N)  [merchant_id]
product (1) ──< chat_session (N) [product_id]
chat_session (1) ──< chat_message (N) [session_id]
user (1) ──< auto_reply_rule (N) [merchant_id]
```

## 自动回复匹配逻辑

```
买家发送消息 (senderRole == CUSTOMER)
    ↓
查询该商家的启用规则 (status=1)，按 priority DESC 排序
    ↓
遍历规则，判断 message.content.contains(rule.keyword)
    ↓
命中第一条 → 插入 SYSTEM 消息 (senderRole=SYSTEM, senderId=0)
未命中 → 不操作
```

## WebSocket 设计（阶段三）

### 连接建立

```
客户端: ws://host/ws/chat?token=eyJhbGciOi...
    ↓
HandshakeInterceptor: 从 URL 参数提取 token → JwtTokenUtils 验证 → 提取 userId
    ↓ 验证通过
WebSocketSession 建立，注册到 OnlineStatusManager
    ↓ 验证失败
拒绝握手，返回 401
```

### 消息推送流程

```
发送方调用 sendMessage (HTTP 或 WebSocket)
    ↓
ChatService 保存消息到数据库
    ↓
检查接收方 OnlineStatusManager
├── 在线 → WebSocket 推送 JSON 消息
└── 离线 → 消息已在库中，对方上线后拉取
```

### WebSocket 消息格式

客户端发送:
```json
{"type": "message", "sessionId": 1, "content": "你好", "contentType": "TEXT"}
{"type": "typing", "sessionId": 1}
{"type": "ping"}
```

服务端推送:
```json
{"type": "message", "data": { "id": 5, "sessionId": 1, "senderId": 2, "senderRole": "MERCHANT", "content": "你好！", "contentType": "TEXT", "createdAt": "..." }}
{"type": "typing", "sessionId": 1, "userId": 2}
{"type": "online", "userId": 2, "status": true}
{"type": "pong"}
```

### 心跳与重连

- 客户端每 30s 发送 `{"type": "ping"}`
- 服务端回复 `{"type": "pong"}`
- 客户端 45s 未收到 pong → 断线重连
- 重连策略: 1s → 2s → 4s → 8s → 16s → 30s（指数退避，上限 30s）

### 在线状态管理

```java
// OnlineStatusManager.java
ConcurrentHashMap<Long, WebSocketSession> onlineSessions;

boolean isOnline(Long userId);
void register(Long userId, WebSocketSession session);
void remove(Long userId);
void sendToUser(Long userId, String message);
Set<Long> getOnlineUserIds();
```

## 前端组件结构

### 买家端

```
ProductDetail.vue / Order.vue
    └── [联系客服] 按钮
            ↓ 点击
        ChatWindow.vue (el-drawer 抽屉)
            ├── 消息列表 (气泡样式，左=对方，右=自己)
            ├── 图片消息渲染 (el-image + 预览)
            ├── 输入框 + 发送按钮
            ├── 图片上传 (复用 FileController)
            └── 轮询/WebSocket 接收新消息
```

### 商家后台

```
AdminLayout 侧边栏
    ├── 客服消息 → ChatList.vue (会话列表)
    │       ├── el-table: 买家名、商品、最后消息、未读数
    │       └── 点击行 → ChatRoom.vue
    │               ├── 同 ChatWindow 的聊天界面
    │               └── 快捷回复按钮
    └── 自动回复 → AutoReplyManagement.vue
            └── el-table + el-dialog: CRUD 关键词规则
```

```
state:
  sessions: []          // 会话列表

  unreadCount: 0        // 全局未读数

mutations:
  SET_SESSIONS / SET_MESSAGES / ADD_MESSAGE
  SET_UNREAD_COUNT / SET_WS_CONNECTED

actions:
  fetchSessions / fetchMessages / sendMessage
  fetchUnreadCount / markAsRead
  startPolling / stopPolling         (阶段二)
  connectWebSocket / disconnectWebSocket  (阶段三)
```
