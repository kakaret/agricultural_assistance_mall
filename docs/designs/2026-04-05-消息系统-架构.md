## 3️⃣ WebConfig 和 JWT 认证

### 3.1 WebConfig 配置

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Resource
    private JwtInterceptor jwtInterceptor;
    
    // 功能1: 统一 /api 路由前缀
    @Override
    public void configurePathMatch(PathMatchConfigurer configurer) {
        configurer.addPathPrefix("/api", clazz ->
            clazz.isAnnotationPresent(RestController.class) &&
            !clazz.getPackage().getName().contains("springfox")
        );
    }
    
    // 功能2: 注册拦截器 (JWT认证)
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(jwtInterceptor)
            .addPathPatterns("/api/**")
            .excludePathPatterns(
                "/api/user/login",
                "/api/user/forget",
                "/api/user/add",
                "/api/user/{id}",
                "/api/email/**",
                "/api/img/**",
                "/api/file/**",
                "/api/doc.html",
                "/api/webjars/**"
            );
    }
}
```

### 3.2 JwtInterceptor - JWT 认证流程

```java
@Component
public class JwtInterceptor implements HandlerInterceptor {
    @Resource
    private UserService userService;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) throws Exception {
        
        // ❶ 获取 Token (支持两种方式)
        String token = request.getHeader("token");              // 方式1: Header
        if (StringUtils.isBlank(token)) {
            token = request.getParameter("token");              // 方式2: Query
        }
        
        // ❷ Token 为空检查
        if (StringUtils.isBlank(token)) {
            response.setStatus(401);
            response.getWriter().print("Token缺失");
            return false;
        }
        
        try {
            // ❸ 解析 JWT，提取用户ID (从 audience 字段)
            String userId = JWT.decode(token).getAudience().get(0);
            User user = userService.getUserById(Integer.parseInt(userId));
            
            if (user == null) {
                response.setStatus(401);
                response.getWriter().print("User not found");
                return false;
            }
            
            // ❹ 验证签名 (⚠️ 关键: 用户密码作为 HMAC-256 密钥)
            // 这意味着:
            // - 登录时: Token = JWT.create().withAudience(userId).sign(user.password)
            // - 验证时: 用 user.password 验证 Token 签名
            // - 密码改变后: 所有旧 Token 失效 ✓
            
            JWTVerifier verifier = JWT.require(Algorithm.HMAC256(user.getPassword()))
                                     .build();
            verifier.verify(token);
            
            LOGGER.info("JWT验证成功，用户: {}", user);
            return true;
            
        } catch (JWTVerificationException e) {
            response.setStatus(401);
            response.getWriter().print("token认证失败，请重新登录");
            return false;
        } catch (Exception e) {
            response.setStatus(401);
            response.getWriter().print("token失效，请重新登录");
            return false;
        }
    }
}
```

**JWT认证流程图:**
```
登录请求:
  POST /api/user/login (username, password)
        ↓
    验证账号密码
        ↓
    生成 Token:
    JWT.create()
       .withAudience(userId)  ← 用户ID
       .sign(Algorithm.HMAC256(password))  ← 用用户密码签名
        ↓
    返回: { token: "eyJ0eXAiOiJKV1QiLCJhbGc..." }

API请求:
  GET /api/product/page
  Headers: { token: "eyJ0eXAiOiJKV1QiLCJhbGc..." }
        ↓
    JwtInterceptor.preHandle()
        ├─ 从 Header 获取 token
        ├─ 解码 JWT，提取 userId (audience)
        ├─ 从数据库查询用户，获取密码
        ├─ 用密码验证签名: Algorithm.HMAC256(password).verify()
        ├─ ✓ 验证成功 → 放行请求
        └─ ✗ 验证失败 → 返回 401
```

```sql
CREATE TABLE `order` (
  `id` BIGINT AUTO_INCREMENT,
  
  -- 外键关系
  `user_id` BIGINT NOT NULL,           -- 买家ID

  -- 退款相关
  `refund_status` TINYINT DEFAULT 0,   -- 0无 1申请 2处理中 3已完成 4失败

  `refund_reason` VARCHAR(255),        -- 退款原因
  
  -- 收货信息
  `recv_address` TEXT,                 -- 收货地址
  `recv_phone` VARCHAR(20),            -- 收货电话
  `recv_name` VARCHAR(50),             -- 收货人名字
  
  -- 时间戳
  `created_at` TIMESTAMP DEFAULT NOW(),
  `updated_at` TIMESTAMP DEFAULT NOW() ON UPDATE CURRENT_TIMESTAMP,
  
  PRIMARY KEY (id),
  FOREIGN KEY (user_id) REFERENCES user(id),
  FOREIGN KEY (product_id) REFERENCES product(id),
  INDEX idx_user_id (user_id),
  INDEX idx_order_user (user_id)
);
```

### 6.1 HTTP 客户端 (request.js)

```javascript
// 创建 axios 实例
const service = axios.create({
    baseURL: process.env.VUE_APP_BASE_API || '/api',
    timeout: 15000
})

// 请求拦截器
service.interceptors.request.use(config => {
    // ❶ 自动添加 Token
    if (store.getters.token) {
        config.headers['Authorization'] = `Bearer ${store.getters.token}`
    }
    
    // ❷ 性能监测
    config.metadata = { startTime: performance.now() }
    
    // ❸ 自动显示加载动画 (仅 POST/PUT/DELETE)
    const isMutation = ['post', 'put', 'delete', 'patch'].includes(config.method?.toLowerCase())
    if (config.showLoading !== false && isMutation) {
        showLoading()
    }
    
    return config
})

// 响应拦截器
service.interceptors.response.use(
    response => {
        return response.data  // 直接返回数据
    },
    error => {
        // ❶ 401 - 登录过期
        if (error.response?.status === 401) {
            Message.error('登录已过期，请重新登录')
            store.dispatch('user/logout')
            router.push('/login')
        }
        
        // ❷ 其他错误处理
        // 404, 500 等...
        
        return Promise.reject(error)
    }
)

export default service
```

## 8️⃣ 关键发现

### ✅ 项目优势

1. **完整的电商流程**
   - ✓ 订单全生命周期管理 (7种状态)
   - ✓ 完善的退款流程 (申请→审核→完成)
   - ✓ 物流实时追踪
   - ✓ 库存精细化管理

2. **灵活的权限体系**
   - ✓ 三层角色模型 (用户/商户/管理员)
   - ✓ 路由级访问控制
   - ✓ JWT Token 认证
   - ✓ 商户数据隔离

3. **现代化的架构**
   - ✓ 前后分离
   - ✓ Spring Boot 3.4.1 最新版本
   - ✓ MyBatis Plus 快速开发
   - ✓ Lombok 代码精简
   - ✓ 统一响应格式

### ⚠️ 需要注意

1. **JWT 认证特殊性**
   - ⚠️ 用户密码作为签名密钥 (密码改变→Token失效)
   - ⚠️ Token 无 TTL (永久有效)
   - ⚠️ 不支持 Token 主动刷新

2. **缺少的功能** (对聊天系统影响)
   - ❌ **没有 WebSocket** (重要!)
   - ❌ 没有消息队列 (可能导致并发问题)
   - ❌ 缺少分布式事务处理

3. **数据库设计**
   - ⚠️ Order 表字段冗余 (收货信息重复)
   - ⚠️ 没有订单快照 (无法追溯历史价格)

```sql
-- 聊天消息表
CREATE TABLE chat_message (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    
    -- 关键关系
    order_id BIGINT,                    -- 关联订单 (可选)
    user_id BIGINT NOT NULL,            -- 消息发送者 (客户)

    is_read TINYINT DEFAULT 0,
    read_at TIMESTAMP,
    
    created_at TIMESTAMP DEFAULT NOW(),
    
    INDEX idx_user_id (user_id),
    INDEX idx_recipient_id (recipient_id),
    INDEX idx_order_id (order_id)
);

-- 聊天会话表
CREATE TABLE chat_session (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    
    user_id BIGINT NOT NULL,            -- 客户ID
    merchant_id BIGINT,                 -- 商户ID (如果是订单相关)
    admin_id BIGINT,                    -- 管理员ID (如果是平台问题)
    
    last_message TEXT,
    last_message_time TIMESTAMP,
    unread_count INT DEFAULT 0,
    
    created_at TIMESTAMP,
    updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_user_id (user_id),
    INDEX idx_merchant_id (merchant_id)
);
```
