# 消息系统实现计划

> 日期: 2026-04-05
> 前置: [designs/2026-04-05-消息系统.md](../designs/2026-04-05-消息系统.md)
> 范围: 阶段二（轮询版客服） + 阶段三（WebSocket 升级）

## 风险评估

| 风险 | 等级 | 缓解策略 |
|------|------|---------|
| WebSocket 与 JWT 拦截器冲突 | 中 | WebSocket 走独立握手认证，不经过 WebConfig |
| 轮询→WebSocket 切换改动大 | 低 | HTTP API 保留作降级，阶段三只加 WebSocket 推送层 |
| 消息量大时查询慢 | 低 | idx_session_time 复合索引 + 分页 |

## 阶段二：客服系统（轮询版）

### Step 1 — 建表 SQL

**文件**: `sql/2026-04-05-chat-tables.sql`

创建 3 张表: chat_session, chat_message, auto_reply_rule

详见 [designs/2026-04-05-消息系统.md](../designs/2026-04-05-消息系统.md) 数据模型章节。

---

### Step 2 — 后端 Entity

**新增 3 个文件**:

| 文件 | 路径 |
|------|------|
| ChatSession.java | `springboot/src/main/java/org/example/springboot/entity/` |
| ChatMessage.java | 同上 |
| AutoReplyRule.java | 同上 |

沿用项目模式: `@Data` + `@TableId(type=IdType.AUTO)` + `java.sql.Timestamp` + `@TableField(exist=false)` 关联对象。

### Step 3 — 后端 Mapper

**新增 3 个文件**:

| 文件 | 路径 |
|------|------|

| ChatMessageMapper.java | 同上 |
| AutoReplyRuleMapper.java | 同上 |

沿用模式: `extends BaseMapper<T>` + `@Mapper`

---

### Step 4 — 后端 Service

**新增 2 个文件**:

| 文件 | 核心方法 |
|------|---------|
| ChatService.java | createOrGetSession, sendMessage, getMessages, getMySessions, getUnreadCount, markAsRead |
| AutoReplyRuleService.java | CRUD + 分页查询 |

关键逻辑:
- `createOrGetSession`: 先查 uk_customer_merchant，有就返回，没有就创建
- `sendMessage`: 插入消息 → 更新 session.last_message_time → 触发自动回复
- 自动回复: senderRole==CUSTOMER 时，查商家规则按 priority DESC，content.contains(keyword) 命中第一条

---

### Step 5 — 后端 Controller

**新增 2 个文件**:

| 文件 | 前缀 | 端点 |
|------|------|------|
| ChatController.java | /chat | POST /session, POST /message, GET /session/{id}/messages, GET /sessions, GET /unread, PUT /message/read/{sessionId} |
| AutoReplyRuleController.java | /auto-reply | POST /, PUT /{id}, DELETE /{id}, GET /page |

## 阶段三：WebSocket 升级

### Step 10 — 后端 WebSocket 基础

**修改**: `springboot/pom.xml` — 添加 `spring-boot-starter-websocket`

**新增 4 个文件**:

| 文件 | 路径 | 说明 |
|------|------|------|
| WebSocketConfig.java | `config/` | @EnableWebSocket, 注册 /ws/chat 端点 |
| WebSocketAuthInterceptor.java | `config/` | 握手时 token 验证 |
| ChatWebSocketHandler.java | `handler/` | 连接/消息/关闭处理 |
| OnlineStatusManager.java | `service/` | ConcurrentHashMap 在线状态 |

**修改**: `ChatService.java` — sendMessage 后检查在线状态，在线则 WebSocket 推送

---

### Step 11 — 前端 WebSocket 客户端

**新增**: `newvue/src/utils/websocket.js`

功能: connect(token), 心跳 30s, 断线重连（指数退避 1s→30s）, onMessage 回调, send, close

---

### Step 12 — 前端组件升级

**修改**: ChatWindow.vue, ChatRoom.vue
- 移除轮询 → 建立 WebSocket
- 收到 message → Vuex ADD_MESSAGE
- 新增在线状态指示器（绿/灰点）
- 新增正在输入提示

**修改**: `store/modules/chat.js`
- 新增: wsConnected, onlineUsers, connectWebSocket, disconnectWebSocket

## 执行顺序

```
阶段二:
  Step 1 (SQL) → Step 2-3 (Entity+Mapper) → Step 4-5 (Service+Controller)
      → Step 6-7 (API+Store) → Step 8 (组件) → Step 9 (路由集成)
      → 验证阶段二

阶段三:
  Step 10 (WebSocket后端) → Step 11 (WebSocket前端) → Step 12 (组件升级)
      → 验证阶段三
```

每个 Step 完成后运行:
- 后端: `cd springboot && mvn compile`
- 前端: `cd newvue && npm run serve`

全部完成后: `cd newvue && npm run test:unit`
