# 🔍 客服聊天系统 - 关键研究发现

## 1️⃣ Entity 类通用模式

所有实体都遵循以下模式：

```
@Data (Lombok)
@TableName (MyBatis Plus)
  ├─ @TableId(type = IdType.AUTO)         # 自增主键
  ├─ @NotBlank / @NotNull (Jakarta)       # 字段验证
  ├─ Timestamp createdAt/updatedAt        # 审计时间
  └─ @TableField(exist = false)           # 关联对象字段
```

**典型实体查看位置:**
- Order.java - 订单（7种状态）
- Product.java - 商品（支持折扣、库存）
- Article.java - 资讯（农技知识）
- User.java - 用户（支持3种角色）
- Logistics.java - 物流追踪

## 2️⃣ Controller 设计规范

**标准 CRUD 接口：**

**关键点：**

## 3️⃣ JWT 认证机制（核心要点）

### 认证流程

```
登录请求 → 生成 Token (用户密码作为签名密钥)
    ↓
发送 Token in Header: token=xxxxx
    ↓
JwtInterceptor.preHandle()
    ├─ 提取 Token 中的 userId (audience 字段)
    ├─ 查询数据库获取用户
    ├─ 用用户密码 + HMAC256 验证签名
    └─ ✓ 通过 → 继续请求
    └─ ✗ 失败 → 返回 401
```

### 重要特性

- ✓ 密码变更 → 旧 Token 全部失效
- ✗ 没有 TTL（永久有效）
- ✗ 不支持 Token 刷新

### 公开接口（无需认证）

```
/api/user/login        ← 登录
/api/user/add          ← 注册
/api/user/forget       ← 忘记密码
/api/user/{id}         ← 获取用户信息
/api/email/**          ← 邮件服务
/api/img/**            ← 图片资源
/api/file/**           ← 文件资源
```

### HTTP 拦截器

```javascript
请求拦截:
├─ 自动添加 Authorization Header
├─ 性能监测
└─ 显示加载动画 (仅 POST/PUT/DELETE)

响应拦截:
├─ 401 → 清除登录状态，跳转登录页
├─ 500 → 显示错误提示
└─ 直接返回 response.data (已处理)
```

## 9️⃣ 需要注意的事项 ⚠️

1. **JWT 特殊性**
   - 密码是签名密钥（密码改→Token失效）
   - 无 TTL（永久有效）
   - 不支持主动刷新

2. **缺少的功能** (对聊天系统影响)
   - ❌ **没有 WebSocket** (需要自己实现!)
   - ❌ 没有消息队列 (高并发可能有问题)
   - ❌ 缺少分布式事务

3. **数据库设计**
   - ⚠️ Order 表信息冗余
   - ⚠️ 没有订单快照

## 🔟 聊天系统集成建议

### 必需表结构

```sql
CREATE TABLE chat_message (
    id BIGINT PRIMARY KEY,
    order_id BIGINT,           -- 可选：关联订单
    user_id BIGINT NOT NULL,   -- 发送者
    recipient_id BIGINT NOT NULL,  -- 接收者
    message_type TINYINT,      -- 0=文本 1=图片 2=文件
    content TEXT NOT NULL,
    is_read TINYINT,
    created_at TIMESTAMP
);

CREATE TABLE chat_session (
    id BIGINT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    merchant_id BIGINT,
    admin_id BIGINT,
    last_message TEXT,
    unread_count INT,
    created_at TIMESTAMP
);
```

### 集成接口

```java
@RestController
@RequestMapping("/chat")
public class ChatController {
    @GetMapping("/messages/{sessionId}")        // 获取聊天记录
    @PostMapping("/message")                    // 发送消息
    @GetMapping("/sessions")                    // 获取会话列表
    @GetMapping("/ws/connect")                  // WebSocket 连接
}
```

### 集成架构建议

1. **继承现有 JWT 认证** → ChatMessage 自动获取 userId
2. **添加 WebSocket 支持** → 实时消息推送
3. **遵循 Controller-Service-Mapper 架构**
4. **使用 Result 统一响应**
5. **考虑消息队列** (RabbitMQ/Kafka) 处理并发
