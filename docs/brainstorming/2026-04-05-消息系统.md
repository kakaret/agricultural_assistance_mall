# 消息系统需求探索

> 日期: 2026-04-05
> 状态: 已确认，进入设计阶段

## 背景

当前农业助农商城缺少买家与商家之间的沟通渠道。买家在浏览商品或下单后无法直接联系商家咨询，商家也无法主动触达买家。

## 目标

实现买家 ↔ 商家的客服聊天系统，分阶段交付：
- 阶段二: 轮询版客服聊天 + 关键词自动回复
- 阶段三: WebSocket 实时通信升级

## 已确认的需求决策

| 决策项 | 结论 | 原因 |
|--------|------|------|
| 商家角色 | User 表 role 字段已有 `MERCHANT` | 无需新增角色体系 |
| 会话模型 | 一对一（买家 ↔ 商家） | 简单够用，不做群聊 |
| 消息类型 | 文字 + 图片 | 图片复用现有 FileController 上传 |
| 阶段节奏 | 阶段二、三紧接着做 | 用户要求连续开发 |
| 自动回复 | 关键词规则匹配 | 先验证需求，不上 AI |
| 消息队列 | 不引入 | 当前规模不需要 |

## 发起场景

- 买家在商品详情页点击「联系客服」→ 与该商品的商家建立会话
- 买家在订单页点击「联系客服」→ 与该订单对应商家建立会话
- 商家在后台查看会话列表 → 回复买家消息

## 边界（不做的事）

- 不做群聊
- 不做视频/语音通话
- 不接入 AI 大模型（后续可扩展）
- 不引入 RabbitMQ/Kafka
- 不修改现有 JWT 认证机制
