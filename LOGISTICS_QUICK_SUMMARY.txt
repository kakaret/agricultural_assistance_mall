🏗️ 后端架构（Java/Spring Boot）
═══════════════════════════════════════════════════════════════════════════════

📦 物流实体 (Logistics.java)
   ├─ id: 物流ID (主键)
   ├─ orderId: 订单ID (外键)
   ├─ receiverName/Phone/Address: 收货信息
   ├─ companyName: 物流公司
   ├─ trackingNumber: 运单号
   ├─ expectedArrivalTime: 预计到达时间
   └─ status: 物流状态 (0待发货, 1运输中, 2已签收, 3已取消)

🎮 物流控制器 (LogisticsController.java)
   ├─ POST   /logistics                    → 创建物流 (管理员发货)
   ├─ PUT    /logistics/{id}/status       → 更新状态
   ├─ GET    /logistics/{id}              → 获取物流
   ├─ GET    /logistics/order/{orderId}   → 根据订单查询 ⭐ (用户查看)
   ├─ GET    /logistics/page              → 分页查询
   ├─ DELETE /logistics/{id}              → 删除物流
   ├─ DELETE /logistics/batch             → 批量删除
   └─ PUT    /logistics/{id}/sign         → 物流签收 ⭐ (确认收货)

💼 物流服务 (LogisticsService.java)
   核心方法：
   ├─ createLogistics()           → 创建物流 + 自动更新订单状态
   ├─ getLogisticsByOrderId()     → 查询订单物流 (用户操作)
   ├─ updateLogisticsStatus()     → 更新物流状态 + 同步订单
   ├─ signLogistics()             → 物流签收 + 订单完成
   ├─ getLogisticsByPage()        → 分页查询 (支持商户过滤)
   ├─ deleteLogistics()           → 删除物流
   └─ deleteBatch()               → 批量删除

🔗 订单服务联动 (OrderService.java)
   ├─ 创建物流时：自动更新订单状态为 2 (已发货)
   ├─ 确认收货时：自动更新物流状态为 2 (已签收)
   ├─ 取消订单时：自动同步物流状态为 3 (已取消)
   └─ 退款完成时：自动同步物流状态为 3 (已取消)

🔄 完整业务流程
═══════════════════════════════════════════════════════════════════════════════

1️⃣ 用户下单支付 
   └─ 订单状态：0 (待支付) → 1 (已支付)

2️⃣ 管理员发货 (触发点：点击"发货"按钮)
   ├─ 打开物流信息填写表单
   ├─ 填写：物流公司、运单号、预计到达时间
   ├─ 调用 API: POST /logistics
   └─ 结果：
      ├─ 物流表插入新记录 (status=1 已发货)
      ├─ 订单状态自动更新为 2 (已发货)
      └─ 用户可看到"查看物流"和"确认收货"按钮

3️⃣ 用户查看物流 (触发点：点击"查看物流"按钮)
   ├─ 打开物流详情对话框
   ├─ 调用 API: GET /order/{orderId}/logistics
   ├─ 后端查询 logistics 表
   └─ 显示物流详情

4️⃣ 用户确认收货 (触发点：点击"确认收货"按钮)
   ├─ 弹出确认对话框
   ├─ 调用 API: PUT /order/{id}/status?status=3
   └─ 结果：
      ├─ 订单状态更新为 3 (已完成)
      ├─ 物流状态自动更新为 2 (已签收)
      └─ 订单完成，用户可申请售后

5️⃣ 用户申请售后 (退货物流)
   ├─ 填写退货物流信息
   ├─ 商家审核
   ├─ 管理员最终处理
   └─ 确认退款或拒绝
